# Why Construct a Circuit File

## Purpose

This example is intentionally small. It shows why a modeler, or an AI generating models, should construct a circuit file instead of leaving a model as loose notes, equations in prose, or an image.

A circuit file is useful because it combines:

- readable documentation
- executable equations
- explicit variables and units
- a visual circuit layout
- reproducible simulation settings

That makes the model easier to inspect, share, debug, and regenerate.

## What This Example Says

The model below is a minimal stock-flow example:

- `YD` is exogenous disposable income
- `Cd` is consumption demand
- `Hs` is saving flow
- `Hh` is household wealth stock

It is not meant to be economically deep. It is meant to be structurally clear.

## Why AI Should Emit Circuit Files

For AI systems, a circuit file is better than free-form explanation because it provides a compact contract:

- the timestep is explicit
- variable names are stable
- equations are machine-readable
- plots, hints, and notes stay attached to the model
- the visual layout can be reconstructed without guessing

That reduces ambiguity and makes later editing much safer.

## Equations

```{r}
Why_Construct_A_Circuit_File <- sfcr_set(
  # [ x=176 y=24 ]
  e1 = YD ~ 100,  # Exogenous disposable income  [mode=param, sliderValue=0 ]
  e2 = \alpha1 ~ 0.8,  # Propensity to consume from current income  [mode=param, sliderValue=0 ]
  e3 = \alpha2 ~ 0.05,  # Propensity to consume from accumulated wealth  [mode=param, sliderValue=0 ]
  e4 = Cd ~ \alpha1 * YD + \alpha2 * last(Hh),  # Consumption demand  [mode=voltage, sliderValue=0 ]
  e5 = Hs ~ YD - Cd,  # Saving flow  [mode=voltage, sliderValue=0 ]
  e6 = Hh ~ last(Hh) + Hs  # Household wealth stock  [mode=param, sliderValue=0 ]
)
```

## Info

```{r}
@info
# Why construct a circuit file?

Use a circuit file when you want one artifact to serve as:

- documentation for humans
- structured input for AI tools
- a simulation-ready model
- a visual explanation of causal structure

In practice, this improves:

- reproducibility: the same file stores parameters, equations, and view state
- auditability: every variable is named and attached to a formula
- debuggability: stock/flow mistakes are visible earlier
- interoperability: parsers can extract equations and metadata directly

Rule of thumb:

- If the model should be rerun, edited, compared, or generated by AI, put it in a circuit file.
- If the model exists only as narrative text, important structure is missing.
@end
```
